## 2. 建议改进（Follow-up，按优先排序）

### N1. 🔴 高优先：CRM 工具错误信息回显内部工具名（违反 PR #14 N1 警告）

- **位置**：`backend/app/agents/tools/crm_tools.py:26`
- **当前**：
  ```python
  def _check_internal(user: User, tool_name: str) -> None:
      if user.is_external:
          logger.warning(...)
          raise PermissionDeniedError(f"External user cannot call {tool_name}")
  ```
- **问题**：错误信息直接回显 `tool_name`（`get_customer_basic` / `list_contracts` / `list_contacts` / `get_service_history` / `find_customer_by_name`），等于把内部工具清单告诉外部用户。攻击者只要触发任意路径让 `build_agent(external_user)` 调到内部工具（理论上铁律 #10 layer 1 物理隔离已阻断，但 defense-in-depth 仍应防），就能通过枚举 `f"External user cannot call X"` 错误信息收集完整内部工具命名清单，配合泄漏的命名规律推断更多内部 endpoint。
- **直接证据**：PR #14 review N1 已经明确警告"raise PermissionError 错误信息不要回显 customer_id / 工具内部细节（防枚举攻击）"，本 PR 派单 prompt §「PR #14 N1 暗坑」段也再次复述。Codex 落地时只接住了 `_check_internal` 二次校验本身，**没接住"错误信息脱敏"那半句**。
- **期望修法**（follow-up commit）：
  ```python
  def _check_internal(user: User, tool_name: str) -> None:
      if user.is_external:
          # 详细信息只写 audit log / loguru，不进 raise message
          logger.warning(
              "External user {} attempted to call internal CRM tool {}",
              user.user_id,
              tool_name,
          )
          # 对外统一错误：不回显工具名 / customer_id / 任何内部识别符
          raise PermissionDeniedError("Permission denied")
  ```
- **依据**：CLAUDE.md 原则 P1（API 响应脱敏，扩展到 PermissionError 消息）+ PR #14 review §2 N1 + ANTIPATTERN K1 精神延伸。
- **配套测试**：`tests/agents/test_crm_tools.py` 应加 `test_external_user_error_message_no_tool_leak`，断言 `str(exc.value) == "Permission denied"`（精确等值，避免未来又退回模糊匹配）。

### N2. 🟡 中：`CRMError.status_code = 502` 对"unknown provider"语义不准

- **位置**：`backend/app/services/crm.py:28-30` 配合 `:144`
- **当前**：`CRMError` 全局 status 502 (Bad Gateway)。`get_crm()` 对 `Unknown CRM_PROVIDER` 也抛同一个错。
- **问题**：502 应该是"下游 vendor 故障"语义。"unknown provider name"是配置错（client 端 / 服务启动期），应该是 500 (Internal Server Error)。混用会让 API 上线后日志难分类（vendor 真挂了 vs 配置错了）。
- **建议**：拆 `CRMError`（502）+ `CRMConfigError`（500），或者在 `get_crm()` 处直接 `raise AppException("Unknown CRM_PROVIDER", status_code=500)`。
- **优先级**：合并到下一个 CRM endpoint PR 顺手做，不阻塞当前。

### N3. 🟡 中：`tool_registry.py` 顶层 assert 前缀清单不全

- **位置**：`backend/app/agents/tool_registry.py:25-37`
- **当前**：`_INTERNAL_ONLY_PREFIXES` 只有 6 个前缀（`get_customer` / `list_contract` / `list_contact` / `get_service` / `find_customer` / `search_docs`），靠模块 import 时 assert 阻止误入 EXTERNAL_TOOLS。
- **问题**：未来如果加 `query_customer_xxx` / `crm_lookup_yyy` 这类新命名（不在前缀里），可以误入 EXTERNAL_TOOLS 不被这个 assert 抓到。#71 snapshot 测试是兜底（snapshot diff 会被发现），但模块 import 层的 best-effort 检查变成"漏网"。
- **建议**：把 `_INTERNAL_ONLY_PREFIXES` 改成 **「外部允许工具白名单」** 反向断言：
  ```python
  _ALLOWED_EXTERNAL_TOOL_NAMES = {"search_external_docs"}
  for _tool in EXTERNAL_TOOLS:
      assert _tool.name in _ALLOWED_EXTERNAL_TOOL_NAMES, (
          f"Tool {_tool.name} not in allowed external whitelist. "
          f"Update _ALLOWED_EXTERNAL_TOOL_NAMES + add prompt injection tests."
      )
  ```
  「白名单」比「黑名单前缀」更安全：新增外部工具必须显式声明，强制 review。
- **优先级**：在 #72 Prompt Injection 测试集落地前做完，避免 #72 实施者踩坑。

### N4. 🟢 低：`build_agent()` 每次新建 LLM + react agent，高并发场景需评估

- **位置**：`backend/app/agents/factory.py:14-17`
- **当前**：`build_agent(user)` 每次调用：(a) `get_llm()` 新建 OpenAILike client（PR #14 N2 已讨论 lru_cache 取舍），(b) `create_react_agent(llm, tools)` 重新组装 LangGraph 状态图。
- **问题**：30 人公司 MVP 场景每用户每会话 1 次，没问题。**但**如果上一轮聊天会话不持久化，每个 query 都 build 一次 agent，单查询多了 50-200ms 的初始化开销 + N 个 token 的 graph 编译。
- **建议**：Phase 2 ACL 中间件落地时同步评估：
  - (a) per-session cache（用 `user.user_id` 做 key，TTL 30 分钟）
  - (b) per-tool-set cache（`tuple(sorted(t.name for t in tools))` 做 key）
- **优先级**：观察到延迟问题再优化，不主动改。

### N5. 🟢 低：`core/audit.py` no-op 只支持 async wrapper

- **位置**：`backend/app/core/audit.py:36`
- **当前**：`async def wrapper(...)` 只能装饰 async 函数。
- **问题**：#69 内 CRM 工具全是 async，OK。但 #70 真实 audit 实现时如果加同步 helper（如 `audit_breach_alert(...)` 同步通知），no-op → 真 audit 切换会断 sync 路径。
- **建议**：#70 spec 加一条"audit 装饰器必须同时支持 sync/async（用 `inspect.iscoroutinefunction(fn)` 分发）"，避免本临时 no-op 限制传染到真实实现。
- **优先级**：#70 落地前必须解决，单独写一条 Handoff §7 备忘。
